Software Design & Development Reflection
How do I approach designing software?
I approach software design by first identifying the problem and defining clear requirements. I then break the system into smaller components and plan how they interact. This includes creating diagrams, outlining data structures, and thinking about user experience. I focus on scalability, readability, and maintainability while choosing appropriate technologies and design patterns.

What new design skills has your work on the project helped you to craft?
Working on the project helped me develop:

Modular thinking (breaking complex systems into manageable parts)
3D spatial reasoning (understanding objects in a 3D environment)
User-focused design (considering how a user interacts with the scene)
Planning before coding (organizing ideas using sketches or pseudocode)


What design process did you follow for your project work?
I followed a structured design process:

Requirement analysis – understanding what the 3D scene needed to include
Planning – sketching scenes and outlining objects and interactions
Design – deciding how to organize code and assets
Implementation – building and integrating components
Testing and refinement – fixing errors and improving visuals


How could tactics from your design approach be applied in future work?
The tactics I used can be reused in future projects by:

Applying step-by-step planning before starting development
Using modular design to simplify debugging and expansion
Iterating on designs gradually instead of aiming for perfection immediately
Keeping usability and clarity in mind


Program Development Reflection
How do I approach developing programs?
I approach development incrementally:

Start with a basic working version
Add features step-by-step
Test frequently
Refactor code to improve quality


What new development strategies did you use while working on your 3D scene?
Some strategies I used included:

Incremental builds (adding one object or feature at a time)
Reusing code patterns for similar objects
Debugging visually (observing scene rendering to find issues)
Experimentation with transformations, lighting, and textures


How did iteration factor into your development?
Iteration was essential. I repeatedly:

Built small parts of the scene
Tested their behavior
Adjusted positioning, scale, and lighting
This process allowed me to gradually improve both functionality and visual quality.


How has your approach to developing code evolved throughout the milestones?
My approach improved significantly:

Early on, I focused on getting code to work
Later, I emphasized clean structure and readability
I became better at anticipating problems and planning ahead
I relied more on testing and refinement, rather than rewriting everything
